## 一、先建立一个核心直觉：变量名只是“对象的标签”

**一句话先懂**：Python 里的变量，更像“贴在对象上的名字”，不是一个真正装值的盒子。

**标准理解**：执行赋值语句时，Python 会把**名字**绑定到某个**对象**上。多个名字可以指向同一个对象；重新赋值时，名字会改绑到别的对象，不是“修改名字本身”。

```python
a = [1, 2]
b = a
b.append(3)
print(a)  # [1, 2, 3]
```

**为什么会这样**：
- `a` 先绑定到列表对象 `[1, 2]`
- `b = a` 不是复制列表，而是让 `b` 也指向同一个列表对象
- `b.append(3)` 改的是那个列表对象本身，所以 `a` 看到的内容也变了

**开发视角**：
- 这就是“共享引用”问题的根源
- 也是 `a = b = []`、浅拷贝、函数默认参数陷阱的共同前提

## 二、普通赋值与多重赋值

### 2.1 普通赋值

**一句话先懂**：用 `=` 把一个值绑定给一个变量名。

**标准语法**：
```python
a = 10
name = 'Tom'
items = [1, 2, 3]
```

**最小例子**：
```python
x = 5
print(x)  # 5
```

### 2.2 多重赋值

**一句话先懂**：一行里同时给多个变量赋值。

**标准语法**：
```python
a, b, c = 1, 2, 3
```

**最小例子**：
```python
name, age, city = 'Alice', 20, 'Shanghai'
print(name, age, city)
```

**关键规则**：
- 左边变量个数要和右边值的个数匹配
- 右边本质上会先形成一个可迭代结构，再按顺序解包给左边

**易错点**：
```python
a, b = [1, 2, 3]   # ValueError：值太多
x, y, z = [1, 2]   # ValueError：值太少
```

### 2.3 链式赋值

**一句话先懂**：多个变量一起绑定到同一个对象。

**标准语法**：
```python
x = y = z = 0
```

**最小例子**：
```python
a = b = 10
print(a, b)  # 10 10
```

**什么时候安全**：
- 右边是不可变对象时，通常没问题，比如整数、字符串、元组

**什么时候危险**：
- 右边是可变对象时，比如列表、字典、集合

```python
list_a = list_b = []
list_a.append(1)
print(list_b)  # [1]
```

#### 为什么 `a = b = []` 会联动（展开理解）

- `[]` 只创建了**一个列表对象**
- `a` 和 `b` 都绑定到这个同一个列表
- 你不是有了“两份空列表”，而是有了“两个名字指向同一份列表”

```python
list_a = list_b = []
print(id(list_a) == id(list_b))  # True
```

**正确写法**：
```python
list_a, list_b = [], []
```

## 三、序列解包

### 3.1 基本解包

**一句话先懂**：把一个序列里的多个值，按顺序拆给多个变量。

**标准语法**：
```python
a, b, c = [10, 20, 30]
x, y = ('left', 'right')
```

**最小例子**：
```python
x, y = [1, 2]
print(x)  # 1
print(y)  # 2
```

**适用对象**：
- 列表 `list`
- 元组 `tuple`
- 字符串 `str`
- 其他可迭代对象

```python
a, b, c = 'ABC'
print(a, b, c)  # A B C
```

### 3.2 嵌套解包

**一句话先懂**：如果数据本身是嵌套结构，左边也可以写成嵌套结构去接。

```python
a, (b, c) = [1, (2, 3)]
print(a, b, c)  # 1 2 3
```

**关键规则**：
- 左右两边不仅“数量”要对齐，**层级结构**也要对齐

```python
# 结构不匹配
# a, (b, c) = [1, 2, 3]  # ValueError / TypeError 场景
```

#### 为什么解包会报 `ValueError`（展开理解）

你可以把解包理解成：
- Python 先看左边需要几个位置
- 再看右边能不能按这个结构提供对应数量的值
- 如果数量不匹配，就报 `ValueError`
- 如果右边压根不是可迭代对象，还会报 `TypeError`

```python
# 值太多
# a, b = [1, 2, 3]

# 值太少
# a, b, c = [1, 2]

# 不是可迭代对象
# a, b = 123
```

## 四、星号解包 `*`

### 4.1 星号收集

**一句话先懂**：`*变量` 会把“多出来的那些值”收集起来。

**标准语法**：
```python
first, *middle, last = [1, 2, 3, 4, 5]
```

**结果**：
- `first = 1`
- `middle = [2, 3, 4]`
- `last = 5`

**关键规则**：
- 一个解包表达式里，通常只允许出现一个 `*`
- `*` 收集到的结果总是**列表**
- `*` 可以放在开头、中间、结尾

```python
*head, last = [1, 2, 3]      # head = [1, 2], last = 3
first, *tail = [1, 2, 3]     # first = 1, tail = [2, 3]
```

#### 为什么星号得到的是列表（展开理解）

因为 `*变量` 的职责不是“保留原类型”，而是“把剩余元素统一收集起来”，Python 用列表作为这个“容器”。

```python
first, *rest = 'abcd'
print(rest)        # ['b', 'c', 'd']
print(type(rest))  # <class 'list'>
```

### 4.2 常见用法

- 忽略中间值：
```python
first, *_ , last = [1, 2, 3, 4, 5]
```

- 只保留前两个：
```python
a, b, *_ = [10, 20, 30, 40]
```

- 切分路径或数据头尾：
```python
first, *body, last = ['header', 'a', 'b', 'footer']
```

- 只有一个带星号的变量时：
```python
*head, = 'abc'
print(head)  # ['a', 'b', 'c']
```

**为什么结尾要加逗号**：
- 左边必须仍然是一个“解包结构”
- `*head,` 表示“把右边所有元素收集到 `head` 里”
- 如果不写逗号，语法结构就不是你想要的这个解包形式

## 五、交换变量为什么不需要临时变量

**一句话先懂**：右边会先整体打包，再统一解包给左边，所以不会“串值”。

```python
a = 10
b = 20
a, b = b, a
print(a, b)  # 20 10
```

#### 为什么不会先把 `a` 改掉导致出错（展开理解）

你可以把它理解成两步：
1. 先计算右边，得到一个临时整体 `(20, 10)`
2. 再把这个整体解包给左边的 `a, b`

所以它不是：
- “先让 `a = b`”
- “再让 `b = a`”

而是一次整体交换。

## 六、字典解包与合并 `**`

### 6.1 调用前先认识 `**`

**一句话先懂**：`**` 表示把字典里的键值对展开。

```python
d = {'name': 'Tom', 'age': 18}
```

在本章先重点掌握两件事：
- 字典合并
- 冲突键覆盖规则

### 6.2 合并字典

```python
dict1 = {'a': 1, 'b': 2}
dict2 = {'b': 3, 'c': 4}
merged = {**dict1, **dict2}
print(merged)  # {'a': 1, 'b': 3, 'c': 4}
```

**关键规则**：
- 后展开的字典，会覆盖前面同名键的值

#### 为什么后者会覆盖前者（展开理解）

因为 Python 是按从左到右的顺序构造新字典：
- 先放入 `dict1` 的键值对
- 再放入 `dict2` 的键值对
- 如果遇到同名键，就用后面的值覆盖前面的值

**开发场景**：
- 合并默认配置和用户配置
- 合并请求参数
- 组合前端传参与后端补充字段

```python
default_config = {'debug': False, 'timeout': 5}
user_config = {'timeout': 10}
config = {**default_config, **user_config}
print(config)  # {'debug': False, 'timeout': 10}
```
