## 一、先建立主线：数据处理通常分三步

**一句话先懂**：大多数数据处理题都可以拆成“先校验 → 再换算 → 最后格式化输出”。

比如：
- 进制转换：先检查基数范围，再换算，再决定输出格式
- 文件大小显示：先拿到原始大小，再按单位阶梯换算，再保留小数位
- 下载时间估算：先确认单位，再做公式计算，再选择秒/分钟/小时展示

## 二、进制表示与转换

<span id="进制表示与转换"></span>
### 2.1 Python 里怎么写二进制、八进制、十六进制

```python
print(0b1010)  # 10
print(0o12)    # 10
print(0xA)     # 10
```

| 进制 | 前缀 | 示例 |
| --- | --- | --- |
| 二进制 | `0b` | `0b1010` |
| 八进制 | `0o` | `0o12` |
| 十六进制 | `0x` | `0xA` |

**一句话先懂**：这些不同写法本质上都只是整数的不同表示形式。

### 2.2 进制转换的“万能思路”

**一句话先懂**：先转十进制，再转目标进制。

这套思路分两步：
- 第一步：`int(字符串, 原进制)`
- 第二步：`bin()` / `oct()` / `hex()` 或者手写“循环取余”

```python
value = int('FF', 16)
print(value)      # 255
print(bin(value)) # 0b11111111
```

### 2.3 从其他进制转十进制

```python
print(int('1010', 2))
print(int('12', 8))
print(int('A', 16))
print(int('FF', 16))
```

**标准语法**：

```python
int(string, base)
```

- `string`：表示数字的字符串
- `base`：这个字符串原本所在的进制

### 2.4 从十进制转二 / 八 / 十六进制

```python
n = 255
print(bin(n))
print(oct(n))
print(hex(n))
```

返回值都是**字符串**，并且带前缀：
- `bin(10)` → `'0b1010'`
- `oct(10)` → `'0o12'`
- `hex(10)` → `'0xa'`

如果你只想保留数字部分：

```python
print(bin(10)[2:])
print(oct(10)[2:])
print(hex(10)[2:])
```

### 2.5 一个自测高频题：把十六进制 `'FF'` 转成二进制字符串

```python
binary_str = bin(int('FF', 16))[2:]
print(binary_str)  # 11111111
```

### 2.6 如果目标不是 2 / 8 / 16 进制怎么办

这时通常用“循环取余法”。

```python
def to_base(n, base):
  digits = '0123456789ABCDEF'
  if n == 0:
    return '0'

  result = []
  while n > 0:
    result.append(digits[n % base])
    n //= base
  return ''.join(reversed(result))

print(to_base(38, 5))  # 123
```

**一句话先懂**：
- `% base` 取当前最低位
- `// base` 去掉当前最低位
- 最后把结果反过来

### 2.7 基数范围为什么常写成 2 到 16

在很多教学或业务题里，只要求处理：
- 二进制到十六进制
- 数字字符 `0-9`
- 字母 `A-F`

所以常常先做边界检查：

```python
if not (2 <= from_base <= 16 and 2 <= to_base <= 16):
  raise ValueError('进制范围必须在 2 到 16 之间')
```

## 三、文件大小格式化：把原始数字变成人能看懂的单位

### 3.1 为什么要做单位格式化

后端或系统里拿到的数据常常是：
- `1536`
- `1048576`
- `3221225472`

如果直接展示，用户不容易一眼看懂。  
所以通常要把它变成：
- `1.50 KiB`
- `1.00 MiB`
- `3.00 GiB`

### 3.2 核心算法：不断除以 1024

```python
def format_file_size(size):
  units = ['B', 'KiB', 'MiB', 'GiB', 'TiB']
  index = 0

  while size >= 1024 and index < len(units) - 1:
    size /= 1024
    index += 1

  return f'{size:.2f} {units[index]}'

print(format_file_size(1536))     # 1.50 KiB
print(format_file_size(1048576))  # 1.00 MiB
```

**一句话先懂**：
- 只要还大于等于 1024，就继续往更高单位换
- 每换一次，单位索引加 1

### 3.3 为什么单位阶梯常写成 `B → KiB → MiB → GiB → TiB`

因为这是按 1024 进制换算的数据单位体系。  
在实际开发里，你也会看到很多人直接写成 `KB / MB / GB`，但严格来说二者有差异。

这里在核心语法阶段，只要先掌握“按 1024 递进换算”的思路就够用了。

## 四、下载时间估算：最容易踩坑的是单位

### 4.1 `MB` 和 `Mbps` 不是一回事

- `MB`：兆字节，常用来表示文件大小
- `Mbps`：兆比特每秒，常用来表示网络带宽

### 4.2 最重要的换算关系

**一句话先懂**：

```python
1 Byte = 8 bits
```

所以如果文件大小是 `MB`，带宽是 `Mbps`，你必须先乘 8 再除。

### 4.3 下载时间公式

```python
download_seconds = file_size_mb * 8 / bandwidth_mbps
```

例如：

```python
file_size_mb = 100
bandwidth_mbps = 20
seconds = file_size_mb * 8 / bandwidth_mbps
print(seconds)  # 40.0
```

### 4.4 为什么很多人会算错

因为会把：
- 文件大小的 `B`
- 带宽里的 `b`

当成同一个单位。

实际上它们差 8 倍。

### 4.5 如何按秒 / 分钟 / 小时更自然地输出

```python
def format_download_time(file_size_mb, bandwidth_mbps):
  seconds = file_size_mb * 8 / bandwidth_mbps

  if seconds < 60:
    return f'{seconds:.1f} 秒'
  if seconds < 3600:
    return f'{seconds / 60:.1f} 分钟'
  return f'{seconds / 3600:.1f} 小时'
```

**一句话先懂**：
- 小于 60：显示秒
- 小于 3600：显示分钟
- 否则：显示小时

这也是自测题里常说的“按什么阈值选输出单位”。

例如：

```python
print(format_download_time(2048, 100))  # 大约 2.7 分钟
```

因为：
- `2048 * 8 / 100 = 163.84` 秒
- 约等于 `2.7` 分钟
- 所以这题通常应该输出“分钟”单位，而不是秒或小时

## 五、防御性编程：很多数据处理题，第一步都应该先做校验

### 5.1 为什么函数第一行常常先校验参数

因为现实输入经常不可靠：
- 可能为空
- 可能是负数
- 可能除数为 0
- 可能基数超范围

所以你写函数时，常常第一步不是算，而是先挡住非法输入。

### 5.2 两个典型边界检查例子

#### 进制范围检查

```python
if not (2 <= base <= 16):
  raise ValueError('base 必须在 2 到 16 之间')
```

#### 下载带宽检查

```python
if bandwidth_mbps <= 0:
  raise ValueError('带宽必须大于 0')
```

#### 文件大小检查

```python
if file_size_mb < 0:
  raise ValueError('文件大小不能为负数')
```

## 六、本章高频问法速记

- **进制转换万能公式是什么**：先 `int(string, 原进制)` 转十进制，再转目标进制。
- **为什么十进制像“中转站”**：因为 Python 内部整数处理最自然，很多转换都可以先归一到十进制。
- **`'FF'` 转二进制怎么写**：`bin(int('FF', 16))[2:]`。
- **文件大小格式化核心算法是什么**：不断除以 `1024`，同时单位向上走一级。
- **`MB` 和 `Mbps` 有什么区别**：一个是字节单位，一个是比特每秒单位。
- **下载时间公式是什么**：`文件大小(MB) * 8 / 带宽(Mbps)`。
- **为什么函数第一行常做边界检查**：先挡住非法输入，避免后面算到一半才报错。
- **常见边界检查有哪些**：基数范围、文件大小不能为负、带宽不能为 0 或负数。

---

**本章小结**：数据处理这一章的关键，不是背几个孤立公式，而是掌握“校验 → 换算 → 格式化输出”这条主线。进制转换、文件大小显示、下载时间估算看起来题型不同，但本质上都在做同一件事：先确认输入合法，再按规则换算，最后把结果用更适合人看的形式表达出来。
