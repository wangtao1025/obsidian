## 一、先建立一个核心观念：Python 很看重“对象能做什么”

### 1.1 一句话先懂

在 Python 里，一个对象常常不需要“显式声明自己属于某接口”，只要它**表现得像**某种对象，就能被当作那种对象使用。

这就是后面“鸭子类型”和“协议”能成立的基础。

### 1.2 什么叫协议（protocol）

可以先把它理解成：
- 某类行为所需要满足的一组方法约定
- 不是一定非要 formal interface 才算

比如：
- 能被 `for` 遍历，通常要符合迭代协议
- 能被 `with` 使用，通常要符合上下文管理器协议
- 能像函数一样调用，通常要实现可调用协议

## 二、鸭子类型（Duck Typing）

### 2.1 一句话先懂

**如果它走起来像鸭子、叫起来像鸭子，那就把它当鸭子。**

翻成 Python 里的话就是：
- 不一定关心对象“名义上是什么类型”
- 更关心它“有没有我需要的行为”

### 2.2 最简单的例子

```python
class Dog:
  def speak(self):
    return 'wang'

class Person:
  def speak(self):
    return 'hello'


def make_sound(obj):
  print(obj.speak())
```

只要传入的对象有 `speak()`，这个函数就能工作。

### 2.3 它和静态类型语言里的接口思路有什么不同

鸭子类型更强调：
- 行为兼容
- 运行时灵活
- 解耦程度更高

代价也很明显：
- 错误可能在运行时才暴露
- 过度灵活会降低可维护性

### 2.4 Python 内置功能里到处都是鸭子类型

常见例子：
- **迭代器协议**：只要对象能按协议提供迭代能力，就能用于 `for`
- **文件对象**：只要有类似 `.read()` / `.write()` 行为，就能被很多代码当作“文件类对象”使用
- **上下文管理器**：只要支持进入 / 退出语义，就能用于 `with`
- **可调用对象**：只要对象能被调用，就能像函数一样用

### 2.5 最佳实践

- 面向“需要的行为”设计函数
- 但不要滥用到完全不做边界约束
- 对外部输入仍要保留必要校验和错误信息

## 三、Python 常见对象协议

### 3.1 迭代协议

相关方法：
- `__iter__()`
- `__next__()`

### 3.2 上下文管理器协议

相关方法：
- `__enter__()`
- `__exit__()`

```python
class Demo:
  def __enter__(self):
    print('enter')
    return self

  def __exit__(self, exc_type, exc, tb):
    print('exit')
```

### 3.3 可调用协议

相关方法：
- `__call__()`

```python
class Greeter:
  def __call__(self, name):
    return f'hello, {name}'
```

### 3.4 序列 / 容器常见协议

相关方法：
- `__len__()`
- `__getitem__()`
- `__contains__()`

### 3.5 字符串表示协议

相关方法：
- `__repr__()`
- `__str__()`

## 四、什么是魔术方法（特殊方法）

### 4.1 一句话先懂

魔术方法就是形如 `__xxx__` 的特殊方法，用来让对象接入 Python 的语法和内置行为。

### 4.2 常见分类

#### 4.2.1 对象创建与生命周期

- `__new__`
- `__init__`
- `__del__`

#### 4.2.2 字符串表示

- `__repr__`
- `__str__`

#### 4.2.3 比较与运算

- `__eq__`
- `__lt__`
- `__le__`
- `__gt__`
- `__ge__`

#### 4.2.4 容器行为

- `__len__`
- `__getitem__`
- `__setitem__`
- `__contains__`

#### 4.2.5 可调用与迭代

- `__call__`
- `__iter__`
- `__next__`

### 4.3 为什么它们重要

因为很多语法糖本质上都会落到这些方法上。
例如：
- `len(obj)` -> `obj.__len__()`
- `obj()` -> `obj.__call__()`
- `for x in obj` -> 迭代协议

## 五、猴子补丁（Monkey Patching）

### 5.1 一句话先懂

猴子补丁就是：**程序运行时，动态修改已有对象、类、模块的属性或方法。**

### 5.2 最简单例子

```python
class User:
  def greet(self):
    return 'hello'


def new_greet(self):
  return 'hi'

User.greet = new_greet
print(User().greet())  # hi
```

### 5.3 它能做什么

- 临时修补行为
- 测试替身 / mock
- 为旧对象动态注入能力

### 5.4 风险在哪里

- 改动隐蔽，可读性差
- 容易影响全局行为
- 团队协作时不好排查

### 5.5 使用建议

- 测试环境可以用，但要克制
- 线上核心逻辑尽量避免随意 monkey patch
- 如果能用更清晰的设计替代，优先用更清晰的设计

## 六、描述符（Descriptor）和属性访问

### 6.1 为什么要知道描述符

因为很多看似“普通属性访问”的语法，底层其实和描述符机制密切相关。

### 6.2 描述符协议的核心方法

- `__get__`
- `__set__`
- `__delete__`

### 6.3 它常见在哪

- `property`
- 方法绑定
- ORM 字段
- 框架里的声明式字段系统

### 6.4 先建立一个直觉就够了

你暂时不必熟练手写复杂描述符，但最好知道：
- Python 的属性访问并不只是“字典查值”
- 它背后有更完整的对象协议系统

## 七、`MethodType` 和“方法绑定”到底是什么

### 7.1 为什么函数放进类里就成了方法

因为当你通过实例访问类中的函数时，Python 会产生“绑定效果”，把实例自动作为第一个参数传入。

### 7.2 `MethodType` 是什么

它来自：

```python
from types import MethodType
```

可以用来把函数显式绑定成某个实例的方法。

```python
from types import MethodType

class User:
  pass


def greet(self):
  return f'hello {self.name}'

u = User()
u.name = 'Tom'
u.greet = MethodType(greet, u)
print(u.greet())
```

### 7.3 这和猴子补丁的关系

某种程度上，它可以被当作一种更明确的“动态加方法”的方式。

## 八、对象、属性、方法查找的大致顺序

### 8.1 初学版直觉

当你访问：

```python
obj.attr
```

Python 不只是“去对象里找一个名字”，它会结合：
- 实例属性
- 类属性
- 描述符
- 继承链

### 8.2 为什么这很重要

因为它直接影响：
- 方法为什么会自动绑定 `self`
- `property` 为什么像属性却能执行逻辑
- 多继承下属性为什么会按 MRO 查找

## 九、给初学者和开发者的共同建议

### 9.1 对初学者

先把这几个概念记住：
- 鸭子类型：看行为
- 魔术方法：接入语法
- 猴子补丁：运行时修改
- 描述符：高级属性协议

### 9.2 对开发者

真正落地时更重要的是：
- 用协议思维理解框架设计
- 用清晰设计替代滥用魔法
- 知道什么时候该动态，什么时候该收敛约束

## 十、本章高频问法速记

- **什么是鸭子类型**：更关注对象是否具备某种行为，而不是只看显式类型。
- **什么是魔术方法**：形如 `__xxx__` 的特殊方法，用来接入 Python 语法和内置行为。
- **什么是猴子补丁**：在运行时动态修改对象、类或模块的属性 / 方法。
- **描述符有什么意义**：它是 Python 属性访问机制的重要组成部分，`property` 等都与之相关。
- **`MethodType` 是什么**：把函数显式绑定为某个实例的方法。

---

**本章小结**：对象模型和协议让 Python 看起来“很灵活”，但它不是乱来的灵活。背后有一整套清晰规则：什么行为对应什么特殊方法，什么访问会触发什么协议，什么动态能力是合理扩展，什么又会变成维护风险。把这些规则看懂后，你就更能理解 Python 的设计味道。
