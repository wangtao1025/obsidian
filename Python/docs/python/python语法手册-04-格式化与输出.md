**学完能做什么**：用 `f-string` 做数字格式化、对齐、补零、码点和进制展示，用 `print()` 控制分隔符、结尾、输出目标和刷新时机，并搞清 `repr()`、`__str__`、`__repr__` 的区别。  
**小白注意**：① `f-string` 本质上是“把值按指定格式变成字符串”。② `print()` 负责“输出到哪里、怎么分隔、结尾加什么”。③ `repr()` 更偏调试，`__str__` 更偏给人看。

## 一、先分清两件事：格式化 和 输出 不是一回事

**一句话先懂**：`f-string` 是先把数据变成你想要的字符串，`print()` 是把这个字符串输出出去。

```python
name = 'Alice'
score = 95.5
text = f'{name} 的成绩是 {score:.1f}'
print(text)
```

上面其实分成两步：
- `f'{name} 的成绩是 {score:.1f}'`：负责格式化
- `print(text)`：负责输出

**开发视角**：
- 你完全可以只格式化，不立刻输出
- 也可以把格式化结果写入文件、日志、接口返回值，而不只是打印到终端

## 二、`f-string`：最常用的格式化方式

**一句话先懂**：`f-string` 就是在字符串前加一个 `f`，然后把变量或表达式写进 `{}` 里。

```python
name = 'Tom'
age = 18
print(f'我叫 {name}，今年 {age} 岁')
```

### 2.1 最基本的写法

```python
x = 10
y = 20
print(f'{x} + {y} = {x + y}')
```

`{}` 里不只能放变量，也能放表达式：
- `{x + y}`
- `{name.upper()}`
- `{len(items)}`

### 2.2 格式说明符怎么读

**标准语法**：

```python
{值:填充字符对齐方式宽度.精度类型}
```

先不用一口气全背住，先抓住最常见的几个部分：
- `宽度`：至少占几位
- `对齐`：左对齐 / 右对齐 / 居中
- `精度`：小数保留几位
- `类型`：按什么方式显示，如十进制、浮点、十六进制

### 2.3 最常用的类型代码

| 类型代码 | 含义 | 示例 |
| --- | --- | --- |
| `d` | 十进制整数 | `f'{42:d}'` |
| `f` | 定点小数 | `f'{3.14159:.2f}'` |
| `x` / `X` | 十六进制 | `f'{255:x}'` → `ff` |
| `b` | 二进制 | `f'{10:b}'` → `1010` |
| `o` | 八进制 | `f'{10:o}'` → `12` |
| `%` | 百分比 | `f'{0.256:.1%}'` → `25.6%` |

```python
n = 255
print(f'{n:d}')
print(f'{n:x}')
print(f'{n:X}')
print(f'{n:b}')
print(f'{n:o}')
```

### 2.4 数字格式化：最常见的是小数位控制

```python
price = 19.9876
print(f'{price:.2f}')  # 19.99
print(f'{price:.1f}')  # 20.0
```

**一句话先懂**：`.2f` 表示按浮点数显示，并保留 2 位小数。

这在开发里非常常见：
- 成绩显示
- 金额展示
- 百分比展示
- 文件大小、时间、统计结果输出

### 2.5 对齐：左、右、居中

```python
text = 'Py'
print(f'|{text:<10}|')
print(f'|{text:>10}|')
print(f'|{text:^10}|')
```

- `<`：左对齐
- `>`：右对齐
- `^`：居中

**开发场景**：
- 打印表格
- 对齐日志输出
- CLI 命令行展示

### 2.6 补零和自定义填充

#### 2.6.1 补零：数字特别常见

```python
n = 42
print(f'{n:05d}')   # 00042
print(f'{n:010d}')  # 0000000042
```

**一句话先懂**：`05d` 表示整数宽度至少 5 位，不足的左边补 `0`。

#### 2.6.2 自定义填充字符

```python
n = 42
print(f'{n:*>6d}')   # ****42
print(f'{n:*<6d}')   # 42****
print(f'{n:*^6d}')   # **42**
```

这里的 `*` 是填充字符，后面的 `<` / `>` / `^` 是对齐方式。

#### 2.6.3 为什么自定义填充时必须写对齐方式

```python
n = 42
print(f'{n:1>6d}')
```

- `1`：填充字符
- `>`：右对齐
- `6`：宽度
- `d`：十进制整数

如果你写成：

```python
# f'{n:16d}'
```

那这里的 `1` 就不会被理解成“填充字符”，而会被理解成宽度的一部分。

**一句话总结**：
- 默认补空格：只写宽度即可
- 补零：常写成 `05d`、`08d`
- 自定义填充：必须写填充字符 + 对齐方式

### 2.7 宽度不够时，不会截断数据

```python
n = 123456
print(f'{n:2d}')  # 123456
```

**一句话先懂**：宽度是“最小宽度”，不是“最大长度”。

也就是说：
- 数据短了，会补空格或补零
- 数据长了，会原样完整显示
- Python 不会为了排版把你的数据切掉

### 2.8 码点、字节与进制展示

#### 2.8.1 码点展示

```python
ch = 'A'
print(f'U+{ord(ch):04X}')  # U+0041
```

- `ord(ch)`：把字符变成 Unicode 码点整数
- `04X`：宽度 4，不足补 0，十六进制大写

#### 2.8.2 字节展示

```python
b = 15
print(f'\\x{b:02x}')  # \x0f
```

- `02x`：宽度 2，不足补 0，十六进制小写

### 2.9 `f-string` 里几个开发者常用技巧

#### 2.9.1 `=` 调试写法

```python
name = 'Tom'
age = 18
print(f'{name=}, {age=}')
# name='Tom', age=18
```

**一句话先懂**：它会把“变量名 + 变量值”一起打印出来，调试很方便。

#### 2.9.2 `!r`：按 `repr()` 方式显示

```python
text = 'apple\torange\n'
print(f'{text}')
print(f'{text!r}')
```

- 普通 `{text}`：更像给人看
- `{text!r}`：更像给程序员调试看，会显示转义字符

## 三、`print()`：控制输出行为

**一句话先懂**：`print()` 不是只会“打印内容”，它还能控制分隔符、结尾、输出目标和刷新时机。

### 3.1 `print()` 的基本签名

```python
print(*objects, sep=' ', end='\n', file=sys.stdout, flush=False)
```

### 3.2 五个高频参数

| 参数 | 默认值 | 作用 |
| --- | --- | --- |
| `*objects` | - | 要输出的内容，可以多个 |
| `sep` | `' '` | 多个对象之间的分隔符 |
| `end` | `'\n'` | 输出结尾追加什么 |
| `file` | `sys.stdout` | 输出到哪里 |
| `flush` | `False` | 是否立即刷新缓冲区 |

### 3.3 最基础用法

```python
print('Hello World')
print(100)
print(True)
print(3.14)
```

也可以一次打印多个值：

```python
name = 'Alice'
age = 25
print('姓名:', name, '年龄:', age)
```

### 3.4 `sep`：控制多个值之间怎么分隔

```python
print('A', 'B', 'C')
print('A', 'B', 'C', sep='-')
print('2026', '03', '06', sep='-')
print('Hello', 'World', sep='')
```

**一句话先懂**：默认是空格，你可以换成任何字符串。

### 3.5 `end`：控制打印结尾是什么

```python
print('Hello', end=' ')
print('World', end='!')
print()
```

默认每次 `print()` 后面都会加一个换行，也就是 `end='\n'`。

#### 3.5.1 不换行输出

```python
print('加载中', end='')
print('...', end='')
print('完成')
```

### 3.6 `sep` 和 `end` 组合起来很常用

```python
print('姓名', '年龄', '城市', sep=' | ', end='\n---\n')
print('张三', '25', '北京', sep=' | ')
```

### 3.7 `file`：不输出到屏幕，也可以输出到文件

```python
with open('output.txt', 'w', encoding='utf-8') as f:
  print('这是写入文件的内容', file=f)
  print('第二行内容', file=f)
```

也可以输出到标准错误：

```python
import sys
print('这是一个错误信息', file=sys.stderr)
```

**开发视角**：
- 调试信息和正式输出可以分开
- 临时写日志、导出文本、保存结果时很方便

### 3.8 `flush`：立刻刷新输出缓冲区

```python
import time

for i in range(3):
  print(i, end=' ', flush=True)
  time.sleep(1)
```

**一句话先懂**：有些输出不是马上显示，`flush=True` 可以强制立刻刷出来。

常见场景：
- 进度条
- 倒计时
- 实时日志

### 3.9 动态更新同一行

```python
import time

for i in range(101):
  print(f'\r处理进度: {i}%', end='')
  time.sleep(0.01)
print('\n完成！')
```

关键点：
- `\r`：回到行首
- `end=''`：不换行

这类写法常见于：
- 下载进度
- 命令行加载动画
- 长任务状态展示

## 四、`repr()`、`__str__`、`__repr__`：给谁看，很重要

### 4.1 `repr()`：更偏调试的“官方表示”

**一句话先懂**：`repr()` 更关注“准确、可调试”，甚至希望看起来能帮助你还原对象。

```python
text = 'apple\torange\n'

print(text)
print(repr(text))
```

输出区别：
- `print(text)`：会真的显示制表符和换行
- `repr(text)`：会把 `\t`、`\n` 显式展示出来

这就是为什么调试字符串时，`repr()` 很有用。

### 4.2 `__str__` 和 `__repr__` 的区别

| 项目 | `__str__` | `__repr__` |
| --- | --- | --- |
| 主要受众 | 普通用户 | 开发者 / 调试 |
| 目标 | 可读、友好 | 准确、明确 |
| 常见触发 | `print(obj)`、`str(obj)` | `repr(obj)`、解释器直接显示、容器展示 |

### 4.3 一个最常见的例子

```python
class Student:
  def __init__(self, name, score):
    self.name = name
    self.score = score

  def __str__(self):
    return f'学生: {self.name} (成绩: {self.score})'

  def __repr__(self):
    return f"Student(name='{self.name}', score={self.score})"

s = Student('Alice', 95)

print(s)
print(repr(s))
print([s])
```

你会看到：
- `print(s)`：优先走 `__str__`
- `repr(s)`：走 `__repr__`
- `print([s])`：容器内部展示也默认看 `__repr__`

### 4.4 为什么容器里常看到 `__repr__`

```python
items = [s]
print(items)
```

这是因为列表、字典、元组这些容器，在显示内部元素时，要尽量保留“可调试的信息”，所以默认倾向使用 `repr()` 风格。

### 4.5 如果没写 `__str__` 会怎样

**一句话先懂**：`print(obj)` 会优先找 `__str__`，如果没有，通常会回退到 `__repr__`。

```python
class User:
  def __repr__(self):
    return 'User(debug-view)'

u = User()
print(u)         # User(debug-view)
print(repr(u))   # User(debug-view)
```

这也是为什么很多类哪怕只实现 `__repr__`，打印出来也还能看。

## 五、本章高频问法速记

- **`f-string` 和 `print()` 的区别**：前者负责把值格式化成字符串，后者负责输出。
- **`.2f` 是什么意思**：按浮点数格式显示，并保留 2 位小数。
- **为什么 `f'{n:05d}'` 能补零**：宽度 5，不足部分在左边补 `0`。
- **为什么自定义填充要写对齐方式**：否则 Python 会把填充字符误当成宽度的一部分。
- **宽度会不会截断数据**：不会，宽度只是最小宽度。
- **`sep` 和 `end` 干什么用**：控制多个对象之间的分隔符，以及输出结尾加什么。
- **`file` 参数有什么用**：把 `print()` 输出到文件或别的流，而不只是终端。
- **`flush=True` 为什么有时必须写**：需要实时看到输出时，避免缓冲延迟。
- **`repr()` 和 `print()` 看字符串为什么不一样**：`repr()` 会显式展示转义字符，更适合调试。
- **`__str__` 和 `__repr__` 的区别**：前者给人看，后者给开发者和容器展示看。

---

**本章小结**：格式化输出这一章，核心不是背多少格式代码，而是建立一个稳定心智：`f-string` 负责“把数据变成合适的文本”，`print()` 负责“把文本送到哪里、以什么样子显示出来”。只要你把这个分工看清，再掌握数字格式、对齐补零、`sep/end/file/flush`、`repr()` 与 `__str__` / `__repr__`，日常开发里绝大多数输出场景就都能处理了。
