## 一、先建立最重要的主线：字符、码点、字节不是同一个东西

**一句话先懂**：人看到的是“字符”，Python 内存里常处理的是“文本对象”，存储和传输时最终落地的是“字节”。

你可以先把这条链记成：

```text
字符 -> Unicode 码点 -> 按某种编码规则变成字节
```

例如 `'中'`：
- 它是一个字符
- 它有一个 Unicode 码点 `U+4E2D`
- 用 UTF-8 编码后会变成三个字节

## 二、编码铁三角：ASCII、Unicode、UTF-8

### 2.1 ASCII 是什么

**一句话先懂**：ASCII 是很早期的一套字符编码标准，主要覆盖英文字母、数字、标点和控制字符。

特点：
- 只能表示很有限的一小批字符
- 英文世界够用
- 但远远不够表示中文、日文、emoji 等全球字符

### 2.2 Unicode 是什么

**一句话先懂**：Unicode 是“给全世界字符统一编号”的大字典。

它做的事情是：
- 规定每个字符对应哪个码点
- 比如 `'A'`、`'中'`、`'😀'` 都有各自的编号

例如：
- `'A'` → `U+0041`
- `'中'` → `U+4E2D`

### 2.3 UTF-8 是什么

**一句话先懂**：UTF-8 是把 Unicode 码点真正编码成字节的一种规则。

也就是说：
- Unicode 解决“编号”问题
- UTF-8 解决“怎么存、怎么传”问题

### 2.4 三者关系一句话总结

- **ASCII**：早期小字符集
- **Unicode**：全世界字符的统一编号体系
- **UTF-8**：把 Unicode 编成字节的具体编码方式

### 2.5 为什么说 UTF-8 和 ASCII 兼容

ASCII 范围内的字符，在 UTF-8 里仍然只占 1 个字节。  
所以很多纯英文文本既可以看作 ASCII，也可以看作 UTF-8。

### 2.6 UTF-8 为什么是“变长编码”

**一句话先懂**：不同字符占的字节数不一样。

常见情况：
- ASCII 字符：通常 1 字节
- 常见汉字：通常 3 字节
- 很多 emoji：通常 4 字节

这也是自测卷会问的重点。

### 2.7 为什么不直接把 Unicode 码点原样存成二进制

至少有两个原因：
- **浪费空间**：很多英文字符如果都按固定大宽度存，会很浪费
- **兼容性差**：UTF-8 对英文和老系统兼容更好，网络和文件场景更常用

## 三、核心操作：`str` 和 `bytes` 之间怎么来回转换

### 3.1 `str` 和 `bytes` 的区别

**一句话先懂**：`str` 是文本，`bytes` 是原始字节数据。

```python
text = '中'
data = b'abc'

print(type(text))
print(type(data))
```

### 3.2 编码：`str -> bytes`

```python
text = '中'
b = text.encode('utf-8')
print(b)  # b'\xe4\xb8\xad'
```

### 3.3 解码：`bytes -> str`

```python
b = b'\xe4\xb8\xad'
text = b.decode('utf-8')
print(text)  # 中
```

### 3.4 一定要分清“谁调用谁”

- 文本对象 `str` 调 `.encode()`
- 字节对象 `bytes` 调 `.decode()`

这两个方向千万别反。

### 3.5 `errors='replace'` 是干什么的

```python
b = b'\xff'
print(b.decode('utf-8', errors='replace'))
```

**一句话先懂**：遇到处理不了的非法字节或非法字符时，不直接报错，而是用替代字符顶上。

常见场景：
- 你想先把脏数据读出来看看
- 不希望整个程序因为个别坏字节直接崩掉

### 3.6 最常见的编码 / 解码报错

```python
# '中'.encode('ascii')  # UnicodeEncodeError
# b'\xff'.decode('utf-8')  # UnicodeDecodeError
```

- `UnicodeEncodeError`：文本转字节失败
- `UnicodeDecodeError`：字节转文本失败

### 3.7 `b'\xe4\xb8\xad'` 和里面的 `\x` 分别是什么

```python
b = b'\xe4\xb8\xad'
print(b)
```

- 这整个 `b'...'` 是一个字节串
- 里面的 `\x` 表示“后面跟着一个十六进制字节值”
- `e4 b8 ad` 这三个字节合起来，正好是 `'中'` 的 UTF-8 编码结果

## 四、`ord()` 和 `chr()`：字符与码点的来回转换

### 4.1 `ord()`：字符转码点整数

```python
print(ord('A'))
print(ord('中'))
```

**一句话先懂**：`ord()` 会告诉你“这个字符在 Unicode 里的编号是多少”。

### 4.2 `chr()`：码点整数转字符

```python
print(chr(65))
print(chr(20013))
```

**一句话先懂**：`chr()` 会把码点整数重新变回字符。

### 4.3 `U+` 表示什么

`U+4E2D` 的意思是：
- 这是一个 Unicode 码点表示法
- `4E2D` 是十六进制
- 它对应的十进制数是 `20013`

### 4.4 如何把字符显示成 `U+4E2D` 这种形式

```python
ch = '中'
print(f'U+{ord(ch):04X}')  # U+4E2D
```

这里结合了：
- `ord()`：拿到码点整数
- `:04X`：转成至少 4 位的大写十六进制

### 4.5 `ord()` 和 `chr()` 最常见用途

- 调试编码问题
- 看字符对应的码点
- 从码点反推出字符
- 生成 `U+xxxx` 形式的可读输出

## 五、UTF-8 字节长度与十六进制展示

### 5.1 UTF-8 里不同字符常见占用字节数

```python
print(len('A'.encode('utf-8')))   # 1
print(len('中'.encode('utf-8')))   # 3
print(len('😀'.encode('utf-8')))   # 4
```

### 5.2 如何得到 `'中'` 的 UTF-8 十六进制形式

```python
print('中'.encode('utf-8').hex())  # e4b8ad
```

这也是自测卷里的高频题。

### 5.3 为什么 `.hex()` 很适合调试字节

因为它可以把难看的原始字节，直接变成更容易肉眼阅读的十六进制字符串。

## 六、本章高频问法速记

- **ASCII、Unicode、UTF-8 的关系是什么**：ASCII 是早期字符集，Unicode 是全局编号体系，UTF-8 是具体编码规则。
- **UTF-8 里 ASCII 字符、汉字、emoji 常见占几字节**：1、3、4。
- **`str` 和 `bytes` 怎么互转**：`str.encode()`，`bytes.decode()`。
- **`errors='replace'` 的作用**：遇到非法内容时用替代字符顶上，避免直接报错。
- **`ord()` 和 `chr()` 分别做什么**：字符转码点、码点转字符。
- **`U+4E2D` 是什么意思**：Unicode 码点的十六进制表示。
- **`b'\xe4\xb8\xad'` 和 `'中'` 的关系**：前者是后者按 UTF-8 编码后的字节结果。
- **为什么不直接把 Unicode 码点原样存成二进制**：空间效率和兼容性都不理想。
- **`'中'` 的 UTF-8 十六进制怎么拿**：`'中'.encode('utf-8').hex()`。

---

**本章小结**：字符编码这一章真正要抓住的只有一条主线：字符本身、Unicode 码点、字节表示是三层不同的东西。只要你分清 ASCII / Unicode / UTF-8 的职责，再熟练掌握 `encode()` / `decode()`、`ord()` / `chr()` 和十六进制字节展示，编码题基本就不会再乱。
