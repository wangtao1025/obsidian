## 零、小白先看：这些符号都是啥？

看教程时经常出现 `U+4E2D`、`b'\xe4\xb8\xad'`、`0x41` 这类写法，容易懵。下面用一句话说清各自含义。

### U+ 是什么？

**U+** 是「**Unicode 码点**」的**十六进制写法**，表示「这个字符在 Unicode 里的编号」。

- **U+0041**：编号是十六进制 0x41（十进制 65），对应字符 `'A'`。
- **U+4E2D**：编号是十六进制 0x4E2D（十进制 20013），对应字符 `'中'`。
- **U+1F600**：编号是十六进制 0x1F600（十进制 128512），对应 Emoji 😀。

所以 **U+ 后面那一串**就是「字符的身份证号」，只不过用**十六进制**写（方便和字节、编码对照）。在 Python 里用 `ord('中')` 得到的是**十进制** 20013，和 U+4E2D 是同一个数，只是写法不同（20013 = 0x4E2D）。

### b'...' 和 \x 是什么？

在 Python 里，**`b'...'`** 表示这是 **bytes（字节）**，不是普通字符串。

- **普通字符串**：`'中'` 是**一个字符**，你看到的是字。
- **字节**：`b'\xe4\xb8\xad'` 是**三个字节**（三个数字），是「中」用 UTF-8 存到硬盘/网络时的真实样子。

**`\x`** 表示「后面两位是**十六进制**，代表一个字节」：

- `\xe4` = 一个字节，十进制 228
- `\xb8` = 一个字节，十进制 184  
- `\xad` = 一个字节，十进制 173

所以 `b'\xe4\xb8\xad'` 就是「三个字节：228, 184, 173」，也就是「中」的 UTF-8 编码。为啥用十六进制？因为一个字节是 0～255，用两位十六进制（00～FF）写最紧凑，文档和调试里都这么写。

### 0x 和 U+ 有什么区别？

- **U+**：**专门给 Unicode 码点用的写法**。来自 Unicode 标准，一看到 U+ 就表示「这是一个字符的码点（十六进制）」。例如 U+4E2D 特指「中」的码点。
- **0x**：**单纯的十六进制数字前缀**，编程语言里通用，不限于码点。可以表示任何十六进制数：颜色（0xFF0000）、内存地址、字节值、或者刚好是某个码点（0x4E2D）都可以。所以 **0x 不是“给 Unicode 用的”，只是“这个数用十六进制写”**；当这个数恰好是码点时，0x4E2D 和 U+4E2D 是同一个数（20013），只是 U+ 强调「这是码点」，0x 只表示「十六进制数」。
- Python 里写 `0x4E2D` 和写 `20013` 等价；U+ 是文档/规范里的写法，代码里一般用 `0x` 或十进制。

### 看到「一堆 e4 b8 ad」怎么读？

- 若写成 **`b'\xe4\xb8\xad'`**：这是 Python 的 **bytes 字面量**，三个字节，每个 `\x??` 一个字节。
- 若写成 **`e4b8ad`** 或 **`E4 B8 AD`**：常说是「**十六进制表示的 UTF-8 字节**」，和上面是同一串数，只是没写 `b'\x...'` 和空格，用 `.hex()` 就能得到：`'中'.encode('utf-8').hex()` → `'e4b8ad'`。

### 对照表（一眼看懂）

| 你看到的写法        | 含义                         | Python 里怎么得到 / 对照       |
| ------------------- | ---------------------------- | ------------------------------ |
| **U+4E2D**          | 字符「中」的 Unicode 码点（十六进制） | `hex(ord('中'))` → `'0x4e2d'`  |
| **20013**           | 同上，十进制                 | `ord('中')` → `20013`          |
| **b'\xe4\xb8\xad'** | 「中」的 UTF-8 编码（3 个字节） | `'中'.encode('utf-8')`         |
| **e4b8ad**          | 上面那 3 个字节的十六进制拼成一串 | `'中'.encode('utf-8').hex()`   |
| **0x41**            | 十六进制数 65（此处恰为 A 的码点） | 代码里直接写 `0x41` 或 `65`    |

记住：**U+ = 专门表示 Unicode 码点（十六进制）**；**0x = 通用十六进制前缀，不限于码点**；**b'\x...' = 一串字节（每个 \x?? 一个字节）**。

## 一、字符编码

### 1.1 编码铁三角的关系真相（自测卷 10.1、10.2 待复习）

1. **ASCII (村级系统)**：早期英文标准。它是“身份证号”与“存储方式”的暴力合一，1 字节解决。
    
2. **Unicode (全球身份证系统/码表)**：
    
    - **本质**：它包含了 ASCII 并扩展了全球所有字符。
        
    - **逻辑**：它只发“码点”（逻辑编号），不负责存。在 0-127 号位上，它与 ASCII 的编号完全对齐。
        
3. **UTF-8 (快递包装系统/编码方案)**：
    
    - **本质**：它是 Unicode 码点转换成二进制字节流的**规则**。
        
    - **特性**：**变长存储**。ASCII 字符占 1 字节（结果与原 ASCII 一模一样），汉字占 3 字节，Emoji 占 4 字节。

**常见疑问：既然 Unicode 已经规定了码点，为什么计算机不直接把码点转成二进制存，非要再转成 UTF-8？**

- **Unicode 只规定了「编号是多少」，没规定「用几个字节存」**。码点可以很大（例如 U+10FFFF），你要存成二进制，就必须先约定：每个字符用几个字节？这就已经是「编码方案」了。
- **若规定「一律用 4 字节存一个码点」**（固定长度）：
  - 英文 'A' 码点 65，也要占 4 字节，大量 0 填充，**非常浪费**。网页、日志、代码里大量英文，体积会暴增。
  - 而且和**老系统不兼容**：以前 ASCII 是 1 字节一个字符，无数文件、协议都是按 1 字节来的，突然变成 4 字节，旧数据全乱套。
- **UTF-8 的做法**：**按需用 1～4 个字节**。英文还是 1 字节（和 ASCII 完全一致），中文 3 字节，Emoji 4 字节。这样既兼容了老 ASCII，又省空间，还能表示全部 Unicode。所以「再转一下」不是为了折腾，而是为了**兼容 + 省空间 + 统一**，三者一起满足。

### 1.2 核心操作：`str` (内存文本) ↔ `bytes` (硬盘字节)（自测卷 10.3、10.4 待复习）

- **编码 (Encode)**：`text.encode('utf-8', errors='replace')`。
    
- **解码 (Decode)**：`data.decode('utf-8', errors='replace')`。
    
    - `errors='replace'`：防止因个别无法识别字符导致的整个系统崩溃。
### 1.3 核心转换工具：`ord()` 与 `chr()`（自测卷 10.5、10.6 待复习）

这是在编码铁三角中，连接 **“字符”** 与 **“Unicode 码点”** 的直达电梯。

| 方法          | 严谨定义 (底层物理)                           | 方便理解 (直觉对照)                                        |
| ----------- | ------------------------------------- | -------------------------------------------------- |
| `ord(char)` | **获取码点**。输入一个字符，返回其对应的十进制 Unicode 编号。 | **“查身份证号”**。看到“中”字，问系统：它的 Unicode 编号是多少？（结果：20013） |
| `chr(code)` | **还原字符**。输入一个十进制编号，返回对应的 Unicode 字符。  | **“按号叫人”**。告诉系统：去把 20013 号对应的字找出来。（结果：“中”）         |

## 二、UTF-8 编码规则（拓展）

### 2.1 编码规则表

| Unicode 范围         | 字节数 | UTF-8 编码格式                         | 数据位数 |
| -------------------- | ------ | -------------------------------------- | -------- |
| U+0000 ~ U+007F      | 1 字节 | `0xxxxxxx`                              | 7 位     |
| U+0080 ~ U+07FF      | 2 字节 | `110xxxxx 10xxxxxx`                     | 11 位    |
| U+0800 ~ U+FFFF      | 3 字节 | `1110xxxx 10xxxxxx 10xxxxxx`            | 16 位    |
| U+10000 ~ U+10FFFF   | 4 字节 | `11110xxx 10xxxxxx 10xxxxxx 10xxxxxx`   | 21 位    |

- **首字节**：`0xxxxxxx` → 1 字节；`110xxxxx` → 2 字节；`1110xxxx` → 3 字节；`11110xxx` → 4 字节。
- **后续字节**：均为 `10xxxxxx`。`x` 为数据位，按码点二进制从低到高填入。

### 2.2 手写实现（原理）

需要理解“码点 → 字节”时，可用位运算实现（日常直接用 `encode`/`decode` 即可）：

```python
def unicode_to_utf8(codepoint):
    if codepoint < 0 or codepoint > 0x10FFFF:
        raise ValueError(f"无效的 Unicode 码点: {codepoint}")
    if codepoint <= 0x7F:
        return bytes([codepoint])
    elif codepoint <= 0x7FF:
        return bytes([0b11000000 | (codepoint >> 6), 0b10000000 | (codepoint & 0x3F)])
    elif codepoint <= 0xFFFF:
        return bytes([
            0b11100000 | (codepoint >> 12),
            0b10000000 | ((codepoint >> 6) & 0x3F),
            0b10000000 | (codepoint & 0x3F)
        ])
    else:
        return bytes([
            0b11110000 | (codepoint >> 18),
            0b10000000 | ((codepoint >> 12) & 0x3F),
            0b10000000 | ((codepoint >> 6) & 0x3F),
            0b10000000 | (codepoint & 0x3F)
        ])
```

### 2.3 简要实例

- **'A'** U+0041：1 字节 → `0x41`
- **'中'** U+4E2D：3 字节 → `0xE4 0xB8 0xAD`
- **'😀'** U+1F600：4 字节 → `0xF0 0x9F 0x98 0x80`

### 2.4 常见问题

- **UTF-8 和 Unicode 区别？** Unicode 是字符集（编号），UTF-8 是编码方式（如何存）。
- **如何判断字节是 UTF-8？** `data.decode('utf-8')` 不抛 `UnicodeDecodeError` 即合法（可封装成 `is_utf8(data)`）。
