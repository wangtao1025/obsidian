## 一、赋值运算符

### 1.1 简单赋值 `=`

**一句话先懂**：把右边的对象绑定给左边的变量名。

```python
x = 10
name = 'Tom'
items = [1, 2, 3]
```

**开发视角**：
- 这里的“赋值”依然不是复制对象，而是绑定
- 所以如果右边是可变对象，多个名字可能共享同一对象

### 1.2 复合赋值运算符

**补充理解**：链式赋值 `a = b = c = 1` 可以理解为“右边的对象先确定下来，再把这个结果绑定给多个名字”。所以这种写法适合不可变对象；如果右边是可变对象，就要小心共享引用问题。

| 运算符 | 常见读法 | 近似展开写法 |
| --- | --- | --- |
| `+=` | 加后赋值 | `x = x + y` |
| `-=` | 减后赋值 | `x = x - y` |
| `*=` | 乘后赋值 | `x = x * y` |
| `/=` | 除后赋值 | `x = x / y` |
| `//=` | 整除后赋值 | `x = x // y` |
| `%=` | 取余后赋值 | `x = x % y` |
| `**=` | 幂后赋值 | `x = x ** y` |

```python
x = 10
x += 5    # 15
x -= 3    # 12
x *= 2    # 24
x /= 4    # 6.0
x //= 3   # 2.0
x %= 2    # 0.0
```

### 1.3 复合赋值最常见的场景

- **计数器**：`count += 1`
- **累加器**：`total += price`
- **字符串拼接**：`message += '!'`
- **列表原地扩展**：`items += [4, 5]`

#### 为什么 `+=` 不一定等价于 `x = x + y`（展开理解）

对不可变对象（如整数、字符串）来说，你可以近似理解为一样：

```python
x = 1
x += 2
```

但对可变对象（如列表）来说，`+=` 往往会**原地修改**：

```python
a = [1, 2]
b = a
a += [3]
print(a)  # [1, 2, 3]
print(b)  # [1, 2, 3]
```

而下面这种写法会创建新列表并重新绑定：

```python
a = [1, 2]
b = a
a = a + [3]
print(a)  # [1, 2, 3]
print(b)  # [1, 2]
```

**开发视角**：
- 对列表、字典、集合等可变对象，要特别注意“原地修改”还是“生成新对象”
- 写共享数据结构时，这一点非常关键

## 二、算术运算符

### 2.1 七种基础算术运算符

| 运算符 | 含义 | 示例 |
| --- | --- | --- |
| `+` | 加法 | `a + b` |
| `-` | 减法 | `a - b` |
| `*` | 乘法 | `a * b` |
| `/` | 除法 | `a / b` |
| `//` | 整除 | `a // b` |
| `%` | 取余 | `a % b` |
| `**` | 幂运算 | `a ** b` |

```python
a = 10
b = 3
print(a + b)   # 13
print(a - b)   # 7
print(a * b)   # 30
print(a / b)   # 3.333...
print(a // b)  # 3
print(a % b)   # 1
print(a ** b)  # 1000
```

### 2.2 `/`：除法

**一句话先懂**：`/` 永远做“普通除法”，结果是浮点数。

```python
print(10 / 2)  # 5.0
print(5 / 2)   # 2.5
```

**小白注意**：哪怕正好除尽，结果仍然是 `float`。

### 2.3 `//`：整除

**一句话先懂**：整除会把结果向下取整。

```python
print(10 // 3)   # 3
print(5 // 2)    # 2
print(10.0 // 3) # 3.0
```

#### 为什么负数时 `//` 看起来“反直觉”（展开理解）

Python 的 `//` 不是“直接去掉小数部分”，而是**向负无穷方向取整**。

```python
print(-10 // 3)  # -4
```

因为：
- `-10 / 3 ≈ -3.333...`
- 向下取整后是 `-4`

### 2.4 `%`：取余

**一句话先懂**：`a % b` 取的是“除完后剩下多少”。

```python
print(10 % 3)   # 1
print(8 % 2)    # 0
```

**高频场景**：
- 奇偶判断：`n % 2 == 0`
- 分页、循环索引
- 时间换算

#### 为什么 `-10 % 3 == 2`（展开理解）

Python 始终保证：

```python
被除数 = 除数 * 商 + 余数
```

并且在除数为正时，余数会落在 `[0, 除数)` 区间里。

```python
print(-10 // 3)  # -4
print(-10 % 3)   # 2
# 因为 -10 = 3 * (-4) + 2
```

所以负数参与时，`%` 的结果可能和直觉不同，但它是严格符合这条公式的。

### 2.5 `**`：幂运算

**一句话先懂**：`a ** b` 表示 `a` 的 `b` 次方。

```python
print(2 ** 3)   # 8
print(9 ** 0.5) # 3.0
```

#### 为什么 `2 ** 3 ** 2` 等于 `512`（展开理解）

因为幂运算符是**右结合**：

```python
2 ** 3 ** 2
# 等价于 2 ** (3 ** 2)
# 等于 2 ** 9
# 等于 512
```

而不是：

```python
(2 ** 3) ** 2  # 64
```

## 三、运算符优先级与结合性

### 3.1 一个够用的优先级顺序

从高到低，先记这一版就够：

1. `()`
2. `**`
3. `+x` / `-x`（正负号）
4. `*` `/` `//` `%`
5. `+` `-`
6. 比较运算
7. `not`
8. `and`
9. `or`

### 3.2 最稳的开发建议：复杂表达式加括号

```python
result = (a + b) * c
```

即使你知道优先级，也建议在：
- 公式稍复杂
- 逻辑判断较多
- 团队协作代码

这些场景里主动加括号，让表达意图更清楚。

### 3.3 三个常见例子

```python
print(2 + 3 * 4)      # 14
print((2 + 3) * 4)    # 20
print(2 ** 3 ** 2)    # 512
```

### 3.4 比较运算会先于逻辑运算

```python
score = 80
team = 'A'
print(score > 60 and team == 'A')  # True
```

这行代码会先算：
- `score > 60`
- `team == 'A'`

然后再把两个比较结果交给 `and` 处理。

### 3.5 一个高频实战：用 `//` 和 `%` 做时间拆分

```python
seconds = 3661
hours = seconds // 3600
minutes = (seconds % 3600) // 60
secs = seconds % 60
print(hours, minutes, secs)  # 1 1 1
```

**一句话先懂**：
- `//` 用来取整部分
- `%` 用来取剩余部分

这类写法在“时间拆分、分页、进制转换”里都很常见。

## 四、除零、浮点与边界问题

### 4.1 除零错误

```python
# 10 / 0
# 10 // 0
# 10 % 0
```

上面三种都会抛出：

```python
ZeroDivisionError
```

**开发建议**：
- 做除法前，先判断分母是不是 `0`
- 参数来自用户输入时，最好先校验

### 4.2 浮点数不是精确十进制

```python
print(0.1 + 0.2)  # 0.30000000000000004
```

这不是 Python 特有问题，而是二进制浮点表示的普遍现象。

**开发建议**：
- 展示给用户时，用格式化控制精度
- 判断是否“几乎相等”时，用容差或 `math.isclose`

## 五、位运算符详解

### 5.1 六种位运算符

| 运算符 | 名称 | 作用 |
| --- | --- | --- |
| `&` | 按位与 | 对应位都为 1 才为 1 |
| `|` | 按位或 | 对应位有 1 就为 1 |
| `^` | 按位异或 | 不同为 1，相同为 0 |
| `~` | 按位取反 | 所有位翻转 |
| `<<` | 左移 | 向左移动若干位 |
| `>>` | 右移 | 向右移动若干位 |

### 5.2 最小例子

```python
x = 12  # 1100
y = 10  # 1010

print(x & y)   # 8
print(x | y)   # 14
print(x ^ y)   # 6
print(~x)      # -13
print(x << 1)  # 24
print(x >> 1)  # 6
```

### 5.3 `~x` 为什么等于 `-x - 1`（展开理解）

Python 整数不是固定 8 位或 32 位，而是按补码规则去解释取反结果。对 Python 整数来说，可以直接记：

```python
~x == -x - 1
```

例如：

```python
~12 == -13
```

### 5.4 高频场景

#### 奇偶判断
```python
n = 15
print((n & 1) == 0)  # False
```

#### 计算 2 的幂
```python
print(1 << 5)  # 32
```

#### 标志位 / 权限掩码
```python
READ = 1    # 001
WRITE = 2   # 010
EXEC = 4    # 100

perm = READ | WRITE
print(perm)  # 3
print((perm & WRITE) == WRITE)  # True
```

#### 用异或交换变量（知道即可）
```python
a, b = 5, 10
a ^= b
b ^= a
a ^= b
print(a, b)  # 10 5
```

**开发建议**：
- 实际项目里交换变量更推荐 `a, b = b, a`
- 位运算更常用于权限、掩码、协议、底层标志位处理
