## 一、这个项目现在在做什么

当前维护的不是“单纯的 Python 面试题整理”，而是一个更完整的笔记体系：

- 主体是 **《Python 全栈开发：核心语法查漏补缺手册》**
- 目标是把 `https://python.docs-hub.com/` 中 66 题及相关扩展主题，**拆成正常学习笔记**，而不是原样搬运题目答案
- 读者定位是：
  - **小白可读**：能顺着看懂语法、概念、示例、易错点
  - **有经验开发者可检索**：能快速按主题查漏补缺

当前项目已经从“只整理核心语法章节”升级为：
- **学习准备**
- **核心语法 13 章**
- **进阶专题补充文档**
- **标准库与第三方库导航**
- **66 题知识点导航**
- **核心语法自测试卷（已含补充专题）**

## 四、当前的写作风格约束

后续如果继续整理，一定要延续下面这套风格，不要跑偏。

### 4.1 不是“题库答案”，而是“学习笔记”

不要写成：
- 面试题站补充
- 题目答案搬运
- 只给结论、不讲规则

要写成：
- 一句话先懂
- 标准语法 / 核心规则
- 最小示例
- 易错点 / 开发建议
- 必要时再补小白向展开解释

### 4.2 同时兼顾小白和专业开发者

原则是：
- **主线短句清晰**，方便检索
- **复杂点单独展开**，但不要整篇都过度口语化
- 既能让新手顺着学，也能让老手快速定位关键词

### 4.3 不要破坏导航可读性

写 Markdown 时要注意：
- 关键标题层级清楚
- 右侧目录能正常显示
- 不要把重要内容都塞在无层级段落里
