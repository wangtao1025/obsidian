## 一、先分清并发和并行

### 1.1 并发（concurrency）

强调的是：
- 多个任务在一段时间内交替推进
- 不一定同一时刻真的同时跑

### 1.2 并行（parallelism）

强调的是：
- 多个任务在同一时刻真正同时执行

### 1.3 为什么这两个词容易混

因为在日常口语里，很多人会混着说。

但在技术语境里最好区分清楚：
- 线程更常被拿来做并发
- 多进程更容易实现 CPU 并行利用

---

## 二、Python 多线程

### 2.1 一句话先懂

多线程是在一个进程里开多个执行线程，让任务交替推进。

### 2.2 更适合什么场景

更常见于：
- 网络请求
- 文件读写
- 爬虫 I/O 等待
- 数据库 / API 调用

也就是 **I/O 密集型任务**。

### 2.3 为什么 Python 多线程经常会提到 GIL

CPython 有 GIL（全局解释器锁）这个实现特点。

初学阶段先记住结论就够：
- 对纯 CPU 密集型任务，多线程往往不是最优选择
- 对 I/O 密集型任务，多线程依然非常常用

### 2.4 最简单例子

```python
from threading import Thread


def task(name):
  print(f'run {name}')

threads = [Thread(target=task, args=(i,)) for i in range(3)]
for t in threads:
  t.start()
for t in threads:
  t.join()
```

## 三、Python 多进程

### 3.1 一句话先懂

多进程是开多个独立进程来执行任务。

### 3.2 更适合什么场景

更常见于：
- 计算密集型任务
- 图像处理
- 批量 CPU 运算
- 需要利用多核 CPU 的场景

### 3.3 它和多线程最大的区别

- 线程共享同一进程内存空间
- 进程彼此更独立

这意味着：
- 线程通信更方便，但共享状态更容易出问题
- 进程隔离更强，但通信成本更高

### 3.4 最简单例子

```python
from multiprocessing import Process


def task(name):
  print(f'run {name}')

processes = [Process(target=task, args=(i,)) for i in range(3)]
for p in processes:
  p.start()
for p in processes:
  p.join()
```

## 四、线程池和进程池

### 4.1 为什么要用“池”

因为频繁手动创建 / 销毁线程和进程也有成本。

“池”的思路是：
- 先准备一组工作者
- 把任务批量提交给它们处理

### 4.2 `concurrent.futures` 是最常见入口

#### 4.2.1 线程池

```python
from concurrent.futures import ThreadPoolExecutor


def square(x):
  return x * x

with ThreadPoolExecutor(max_workers=4) as executor:
  result = list(executor.map(square, [1, 2, 3, 4]))
```

#### 4.2.2 进程池

```python
from concurrent.futures import ProcessPoolExecutor
```

用法很像，只是底层执行单元变成进程。

### 4.3 为什么线程池 / 进程池更常见

因为它们通常比手动管理：
- 更简洁
- 更稳定
- 更适合批量任务提交

## 五、怎么选：线程、进程、线程池、进程池

### 5.1 I/O 密集型

优先考虑：
- 多线程
- 线程池

### 5.2 CPU 密集型

优先考虑：
- 多进程
- 进程池

### 5.3 如果任务量很多、结构类似

优先考虑“池”。

### 5.4 如果只是学习机制

可以先手写线程 / 进程，再过渡到池。

## 六、共享状态和通信问题

### 6.1 线程为什么容易出共享状态问题

因为多个线程通常共享同一个进程内存。

这会带来：
- 竞争条件
- 数据不一致
- 需要锁

### 6.2 进程如何通信

常见方式有：
- Queue
- Pipe
- Manager
- 序列化传输

### 6.3 初学者先记住的原则

并发不只是“开多个执行单元”，更重要的是：
- 数据怎么共享
- 状态怎么同步
- 资源怎么安全回收

## 七、和 `asyncio` 的关系怎么理解

虽然这题站重点是线程 / 进程 / 线程池，但你最好知道：
- Python 还有一大类并发方式叫异步 I/O（`asyncio`）
- 它特别适合大量 I/O 等待场景

不过如果你现在先学线程 / 进程，这个认知就够了。

---

## 八、本章高频问法速记

- **并发和并行的区别**：一个强调任务交替推进，一个强调同一时刻真正同时执行。
- **多线程适合什么**：I/O 密集型任务。
- **多进程适合什么**：CPU 密集型任务。
- **为什么 Python 多线程常提 GIL**：因为 CPython 中它会影响 CPU 密集型线程并行效果。
- **线程池 / 进程池的意义**：复用执行单元，适合批量提交同类任务。

---

**本章小结**：并发的核心不是“会开几个线程”，而是知道任务类型、执行单元和状态管理该怎么匹配。抓住 `I/O 用线程，CPU 用进程，批量任务优先用池` 这条主线，你就已经建立了并发入门最有用的判断框架。
