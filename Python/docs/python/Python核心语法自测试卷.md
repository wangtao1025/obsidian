## 赋值与算术运算符

- **1** 简单赋值运算符是 `=`。两种写法：① `a, b, c = 10, 20, 30`（多重赋值）；② `data = [10, 20, 30]`；`a, b, c = data`（序列解包）。

- **2** `+=` 等价于 `x = x + y`，其余类推：`-=`、`*=`、`/=`、`//=`、`%=`、`**=` 分别为减、乘、除、整除、取余、幂后再赋值。例：`x += 5` 即 `x = x + 5`。

- **3** 依次为：`10` → `15` → `12` → `24` → `6.0`。

- **4** 计数器：`counter += 1`；累加器：`total += num`；字符串拼接：`message += " World"`；列表扩展：`my_list += [4, 5]` 或 `my_list *= 2`。

- **5** 加 `+`、减 `-`、乘 `*`、除 `/`、整除 `//`、取余 `%`、幂 `**`。整除为向下取整，取余为求余数，幂为次方。

- **6** `10 / 3` 结果为浮点型 `3.333...`；`10 // 3` 为 `3`；`10 % 3` 为 `1`。

- **7** `2 ** 3` 为 `8`；`2 ** 3 ** 2` 等价于 `2 ** (3 ** 2) = 2 ** 9 = 512`（幂右结合）；`-10 // 3` 为 `-4`；`-10 % 3` 为 `2`。

- **8** ① 除零会抛出 `ZeroDivisionError`，需 try-except；② 浮点运算有精度误差，比较用 `math.isclose()`；③ 负数整除/取余在不同语言中规则不同，Python 中余数符号与被除数一致等。

- **9** `hours = total_seconds // 3600`；`minutes = (total_seconds % 3600) // 60`；`seconds = total_seconds % 60`。

- **10** 从高到低：幂 `**` → 一元 `+x -x ~x` → 乘除取余 `* / // %` → 加减 `+ -` → 比较 → `not` → `and` → `or` → 赋值。口诀：算术 > 比较 > 逻辑 > 赋值；幂最高，赋值最低。

- **11** 先算 `c ** 2`，再 `b * (c**2)`，最后 `a + ...`。先算 `a + b` 再乘：`(a + b) * (c ** 2)`。

- **12** 结合性指同优先级时从左到右或从右到左计算。`**` 右结合，故 `2 ** 3 ** 2 = 2 ** (3**2) = 512`。赋值链从右向左：先 `c = 1`，再 `b = c`，再 `a = b`。

- **13** 输出 `16`。先算 `2 * 3 = 6`，再执行 `d = d + 6`，即 `10 + 6 = 16`（复合赋值在算术之后执行）。

- **14** 括号能明确计算顺序，避免因记错优先级导致逻辑错误，并提升可读性、便于他人维护。

- **15** 先执行比较运算（`score > 60` 和 `team == "A"`），再执行逻辑与 `and`。比较优先级高于逻辑。

- **16** 抛出 `ZeroDivisionError`。可用 `result = a / b if b != 0 else None` 或 try-except 捕获后赋 `None`。

- **17** 浮点数用二进制近似存储，部分十进制小数无法精确表示。推荐用 `math.isclose(a, b)` 或 `abs(a - b) < 1e-9` 判断"足够接近"。

- **18** 圆面积：`area = pi * r ** 2`。欧氏距离：`distance = ((x2-x1)**2 + (y2-y1)**2)**0.5`。

- **19** `my_list += [4, 5]` 原地修改列表，`my_list` 仍指向原对象。`my_list = my_list + [4, 5]` 会创建新列表并赋给 `my_list`，若其它引用仍指向原列表则不受影响。

- **20** 赋值与算术中：算术高于赋值；算术内部幂最高，其次乘除取余，最后加减；同级多数左结合，幂与赋值为右结合；不确定时用括号。

- **21** `a + b * c ** b` = `2 + 3 * (4**3)` = `2 + 3*64` = `194`。`(a + b) * (c ** b)` = `5 * 64` = `320`。

- **22** 除零：用条件判断或 try-except；浮点精度：比较用 `math.isclose()` 或容差；优先级混淆：多用括号明确顺序。

## 位运算符

- **1** 六种：`&` 按位与（对应位都为 1 结果为 1）、`|` 按位或（有 1 则 1）、`^` 按位异或（不同为 1）、`~` 按位取反（所有位取反）、`<<` 左移（低位补 0）、`>>` 右移（高位按规则补）。  
- **2** `12 & 10` = 8，`12 | 10` = 14，`12 ^ 10` = 6，`~12` = -13。Python 中 `~x` 等价于 `-x - 1`。  
- **3** 左移 n 位等价于乘 2^n，右移 n 位等价于整除 2^n。`12 << 2` = 48，`12 >> 2` = 3。  
- **4** 正数右移很多位会变成 0（相当于一直整除 2）。负数不会变成 0，Python 是算术右移，会趋向 -1。  
- **5** `(n & 1) == 0` 为偶数，`(n & 1) == 1` 为奇数。原理：二进制最低位为 0 即偶数，为 1 即奇数，与 1 做按位与即得最低位。  
- **6** `1 << n` 即 2 的 n 次方。例：`1 << 5` = 32。  
- **7** `a ^= b`；`b ^= a`；`a ^= b`。（或 `a, b = a ^ b, a` 等等价写法）  
- **8** 添加权限：`current | permission`（按位或）；移除权限：`current & ~permission`（与取反后的掩码与）；检查权限：`(current & permission) == permission`（按位与后判断）。  
- **9** 位运算对整数的每一位操作，返回整数；逻辑运算对表达式真假求值，返回布尔或操作数（and/or 可返回操作数）。  
- **10** `0 10 20 5`（x&1=0 偶，x|2=10|2=10，x<<1=20，x>>1=5）。  
- **11** 不会。Python 整数无固定位宽，左移很多位只会得到很大的整数。C/Java 等有位数限制会溢出。  
- **12** `x ^ 0 == x`（与 0 异或等于自身）；`x ^ x == 0`（与自身异或等于 0）。

- **13** ① **r 开头**（r、r+、rb、r+b）：文件必须已存在，不存在会报错；不会清空原内容。② **w 开头**（w、w+、wb、w+b）：文件不存在就创建，存在则先清空再写。③ **a 开头**（a、a+、ab、a+b）：文件不存在就创建，存在不清空，只在末尾追加。

- **14** 日常用的字符串模式（如 `"r"`、`"w+"`）在底层会被转换成系统能识别的位标志再传给操作系统；`os.open()` 则直接使用这些位标志（如 `os.O_RDONLY`、`os.O_WRONLY | os.O_CREAT | os.O_TRUNC`），用按位或 `|` 把多个选项组合成一个整数。所以说“文件模式在底层是位组合”——多种开关（读/写/追加/二进制等）各占不同二进制位，用 `|` 组合后一次传入，与权限掩码的思路一致。

- **39** 一种合理回答：① 分支很少、逻辑简单 → 用 `if-elif-else`，直观清晰；② 纯“值映射”且分支较多 → 用字典映射，查找 O(1)，易于新增/修改分支；③ 每个分支逻辑较复杂 → 用函数字典，把每个分支写成独立函数，主流程更干净，便于维护和测试。

- **40** 大量分支时，`if-elif-else` 是从上到下依次判断，最坏情况时间复杂度 O(n)；字典映射通过哈希表查找，平均时间复杂度 O(1)，在“根据键选择结果”的场景下更高效。工程实践中，字典映射不仅性能更好，还能把映射关系集中写在一处，新增/删除分支只需改字典，代码结构更清晰，特别适合配置选择、命令路由、状态机等场景。
