## 一、什么是面向对象编程（OOP）

### 1.1 一句话先懂

面向对象编程，就是把“数据”和“处理数据的行为”组织到对象里。

### 1.2 为什么要这样做

因为真实项目里，很多东西天然适合被建模成“对象”：
- 用户
- 订单
- 商品
- 文件
- 数据库连接

### 1.3 Python 里讲 OOP，不要背成教条

Python 支持面向对象，但它并不要求你“所有代码都必须写成类”。

更合理的理解是：
- 需要建模、封装状态时，用类很合适
- 简单流程、简单工具函数，函数式写法也很好

---

## 二、类和对象

### 2.1 类是什么

类是“对象的模板”。

### 2.2 对象是什么

对象是按类创建出来的具体实例。

```python
class User:
  pass

u = User()
print(type(u))
```

### 2.3 类和对象的关系怎么记

- 类：图纸
- 对象：按图纸造出来的具体东西

---

## 三、`self` 到底是什么

### 3.1 一句话先懂

`self` 表示“当前这个实例对象本身”。

```python
class User:
  def say_hi(self):
    return 'hi'
```

当你写：

```python
u = User()
u.say_hi()
```

本质上相当于：

```python
User.say_hi(u)
```

### 3.2 为什么第一个参数要写 `self`

因为实例方法需要知道“现在操作的是哪个对象”。

### 3.3 `self` 只是约定名吗

是的，语法上你可以写别的名字。
但工程里几乎都写 `self`，不要故意搞特殊。

## 四、构造初始化：`__init__` 和 `__new__`

### 4.1 `__init__`：初始化对象

```python
class User:
  def __init__(self, name):
    self.name = name
```

它最常用于：
- 给实例属性赋初值
- 建立对象初始状态

### 4.2 `__new__`：真正创建对象

`__new__` 比 `__init__` 更底层。
通常：
- `__new__` 负责创建对象
- `__init__` 负责初始化对象

大多数基础开发里，你只需要熟练掌握 `__init__`。

### 4.3 什么时候才需要关注 `__new__`

通常是：
- 不可变类型子类化
- 单例模式
- 更底层的对象创建控制

## 五、类变量、实例变量、方法类型

### 5.1 实例变量

属于每个对象自己：

```python
class User:
  def __init__(self, name):
    self.name = name
```

### 5.2 类变量

属于整个类共享：

```python
class User:
  count = 0
```

### 5.3 实例方法、类方法、静态方法

#### 5.3.1 实例方法

最常见，默认第一个参数是 `self`。

#### 5.3.2 类方法

用 `@classmethod`，第一个参数通常是 `cls`。

```python
class User:
  total = 0

  @classmethod
  def get_total(cls):
    return cls.total
```

#### 5.3.3 静态方法

用 `@staticmethod`，不自动接收 `self` 或 `cls`。

它更像“放在类命名空间里的普通函数”。

## 六、封装与“私有”

### 6.1 Python 的封装更强调约定

Python 不是那种极端强制的私有机制风格，它更强调：
- 先靠命名约定表达边界
- 再根据需要做更严格控制

### 6.2 单下划线 `_name`

通常表示：
- 内部使用
- 外部最好不要直接依赖

### 6.3 双下划线 `__name`

会触发名称改写（name mangling），主要目的是：
- 减少子类里不小心覆盖同名属性的风险

但它不是“绝对不可访问”的安全机制。

### 6.4 更 Pythonic 的封装方式

很多时候，真正常用的是：
- 合理命名
- 属性访问约定
- `property`

## 七、继承、重写和 `super()`

### 7.1 单继承

```python
class Animal:
  def speak(self):
    return '...'

class Dog(Animal):
  def speak(self):
    return 'wang'
```

### 7.2 方法重写（override）

子类定义了和父类同名方法，就会覆盖父类实现。

### 7.3 `super()` 是什么

`super()` 用来在子类中调用父类（更准确说是 MRO 下一个类）的实现。

```python
class Animal:
  def __init__(self, name):
    self.name = name

class Dog(Animal):
  def __init__(self, name, age):
    super().__init__(name)
    self.age = age
```

### 7.4 `isinstance()` 和 `issubclass()`

```python
print(isinstance(Dog(), Animal))
print(issubclass(Dog, Animal))
```

这两个在 OOP 题里很高频。

## 八、多态与鸭子类型在 OOP 里的位置

### 8.1 多态怎么理解

不同对象，对同一个调用接口，表现出不同实现。

### 8.2 Python 的多态为什么和鸭子类型关系很近

因为 Python 不一定要求你显式继承同一个父类，只要对象行为兼容，很多场景就能一起工作。

这也是 Python 面向对象风格比较灵活的原因。

---

## 九、Python 是否支持多重继承

### 9.1 结论先说

Python **支持多重继承**。

```python
class A:
  pass

class B:
  pass

class C(A, B):
  pass
```

### 9.2 多重继承最容易出什么问题

最典型的是：
- 同名方法来自多个父类
- 查找顺序容易让人迷糊

### 9.3 什么是 MRO

MRO 是 **方法解析顺序**。

你可以理解成：
- 当属性 / 方法需要沿继承链查找时，Python 按什么顺序找

可以通过：

```python
print(C.__mro__)
```

来看。

### 9.4 为什么 `super()` 在多继承里更重要

因为它能帮助各个类沿着 MRO 协作，而不是写死某个父类名。

### 9.5 多继承一定好吗

不一定。

建议：
- 能用组合解决时，优先考虑组合
- 多继承只在确实有清晰设计价值时再用

## 十、组合 vs 继承：真实项目里怎么选

### 10.1 继承适合什么

- “is-a” 关系明确
- 子类确实是父类的一种

### 10.2 组合适合什么

- “has-a” 关系更强
- 希望降低耦合
- 不想陷入复杂继承树

很多实际项目里，**组合往往比深层继承更稳**。
