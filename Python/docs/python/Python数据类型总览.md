## 一、最常见的数据类型总表

| 类型 | 例子 | 类型定位 | 是否可变 | 常见用途 |
| --- | --- | --- | --- | --- |
| `int` | `10` | 整数 | 否 | 计数、索引、数量 |
| `float` | `3.14` | 浮点数 | 否 | 小数、比例、测量值 |
| `bool` | `True` / `False` | 布尔值 | 否 | 条件判断 |

## 二、按“值的性质”来分：可变与不可变

### 2.1 可变对象

常见可变对象：
- `list`
- `dict`
- `set`

可变对象的特点是：
- 对象创建后，内容还能原地修改
- 多个变量引用同一对象时，修改会彼此可见

```python
nums = [1, 2]
alias = nums
nums.append(3)
print(alias)  # [1, 2, 3]
```

### 2.2 不可变对象

常见不可变对象：
- `int`
- `float`
- `bool`
- `str`
- `tuple`
- `bytes`
- `None`

**一句话先懂**：不可变对象不是“不能重新赋值”，而是“不能原地改内容”。

```python
name = 'Tom'
name = name.upper()
```

这里不是把原字符串改了，而是产生了新字符串，再让 `name` 指向它。

## 三、按结构来分：数值、序列、映射、集合

### 3.1 数值类型

- `int`
- `float`
- `bool`

注意：`bool` 在底层上是 `int` 的子类，但在学习和开发表达上，通常把它看成逻辑类型更自然。

### 3.2 序列类型

有顺序、可按位置访问：
- `str`
- `list`
- `tuple`
- `bytes`

这些类型通常支持：
- 索引
- 切片
- `len()`
- 遍历

### 3.3 映射类型

- `dict`

它不是按位置访问，而是按 **键** 访问。

### 3.4 集合类型

- `set`

它的重点不是顺序，而是：
- 元素唯一
- 成员判断快
- 支持交并差等集合运算

## 四、`None` 是什么，它为什么单独重要

### 4.1 一句话先懂

`None` 表示：
- 没有值
- 结果为空
- 这个位置现在不适用

### 4.2 它不是这些东西

- 不是 `0`
- 不是空字符串 `''`
- 不是空列表 `[]`
- 不是 `False`

```python
print(None == False)  # False
print(None == 0)      # False
```

### 4.3 开发里最常见的判断方式

```python
if value is None:
  print('没有值')
```

通常推荐用 `is None`，而不是 `== None`。

---

## 五、`str` 和 `bytes`：文本和字节不要混着想

### 5.1 `str` 是什么

`str` 表示“文本”，也就是你平时看到的人类可读字符。

### 5.2 `bytes` 是什么

`bytes` 表示“字节序列”，更接近文件、网络、底层传输。

### 5.3 它们之间如何转换

```python
text = '你好'
data = text.encode('utf-8')
print(data)
print(data.decode('utf-8'))
```

开发里常见规则：
- 进入程序后尽量当 `str` 处理
- 读写文件、网络传输时再和 `bytes` 互转

## 六、什么是“可哈希”，为什么字典键有要求

### 6.1 一句话先懂

如果一个对象可以当作字典键或集合元素，通常就要求它是 **可哈希的**。

### 6.2 一般规律

通常：
- 不可变对象更可能可哈希
- 可变对象通常不可哈希

例如：
- `str` 可以当字典键
- `tuple` 在元素也都可哈希时，可以当字典键
- `list` 不能当字典键
- `dict` 不能当字典键

```python
data = {('a', 1): 'ok'}
# {[1, 2]: 'bad'}  # TypeError
```

### 6.3 为什么会这样

因为字典和集合依赖哈希值来定位元素。
如果一个对象内容会变，哈希值就可能失去稳定性。

## 七、`type()` 和 `isinstance()` 怎么用

### 7.1 `type()`：看当前对象的直接类型

```python
print(type(10))
print(type([1, 2, 3]))
```

### 7.2 `isinstance()`：更常用于实际开发

```python
print(isinstance(10, int))
print(isinstance(True, bool))
```

为什么很多时候更推荐 `isinstance()`：
- 它更贴近“这个对象能不能按某类东西用”
- 会考虑继承关系

## 八、类型转换：最常见的几种写法

```python
int('123')
float('3.14')
str(100)
list('abc')
tuple([1, 2, 3])
set([1, 1, 2])
dict([('name', 'Tom')])
```

### 8.1 转换时要注意什么

- `int('12.3')` 会报错
- `list('abc')` 会变成 `['a', 'b', 'c']`
- `set()` 会去重
- `dict()` 需要是键值对结构

## 九、开发里最重要的“选类型”直觉

### 9.1 需要顺序，还要频繁增删改

优先考虑：`list`

### 9.2 结构固定，不希望被误改

优先考虑：`tuple`

### 9.3 需要按名字取值

优先考虑：`dict`

### 9.4 需要去重或快速成员判断

优先考虑：`set`

### 9.5 处理文本

优先考虑：`str`

### 9.6 处理二进制数据

优先考虑：`bytes`

## 十、初学者最容易混淆的几组区别

### 10.1 `list` 和 `tuple`

- 都有顺序
- 都能索引和切片
- 列表可变，元组不可变

### 10.2 `str` 和 `list`

- 都是序列
- 字符串不可变
- 列表通常装的是多个独立元素

### 10.3 `dict` 和 `set`

- `dict` 是键值对
- `set` 只有元素，没有值

### 10.4 `==` 和 `is`

- `==` 比较值是否相等
- `is` 比较是不是同一个对象

这个知识点会在后面的身份与成员章节里系统展开。

## 十一、本章高频问法速记

- **Python 常见数据类型有哪些**：数值、文本、字节、序列、集合、映射和 `None`。
- **哪些是可变对象**：`list`、`dict`、`set`。
- **哪些是不可变对象**：数字、字符串、元组、字节、`None`。
- **什么是可哈希**：能稳定参与哈希计算，通常可作为字典键或集合元素。
- **`type()` 和 `isinstance()` 的区别**：一个更偏直接类型查看，一个更偏实际类型判断与继承关系。

---

**本章小结**：学习数据类型，不要只背“有哪些类型”，更要建立三条主线：它是不是有顺序、它能不能改、它适不适合当前任务。只要把“可变 / 不可变”“序列 / 映射 / 集合”“文本 / 字节”这三套区分建立起来，后面的大部分语法就不容易乱。
